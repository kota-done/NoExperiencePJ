# 将来改修案メモ

## 認証方式と鍵管理の見直し

### 現状
- ローカル学習環境では、Spring Boot アプリから MySQL 8 系へ接続するために `allowPublicKeyRetrieval=true` を採用している。
- これは JDBC ドライバが MySQL サーバの公開鍵取得を許可するための設定であり、ローカル開発用途の暫定対応として採用している。

### 採用理由
- Docker 化した MySQL 8 系コンテナに対して、アプリが追加の複雑な認証設定なしで接続できるようにするため。
- 未経験エンジニア向け学習プロジェクトとして、初期学習時の構築難易度を上げないため。

### 課題
- アプリ側が公開鍵の動的取得に依存している。
- 認証方式や鍵管理の責任がアプリ側設定に寄っている。
- 将来の運用統制や本番相当構成への移行を考えると、構成が暫定的である。

### 改修案
- アプリ側で `allowPublicKeyRetrieval=true` に依存しない構成へ移行する。
- Docker 側で MySQL の認証方式を明示的に制御する。
- Docker 側で鍵・証明書を管理し、アプリ側は動的な公開鍵取得を行わない構成を検討する。
- 必要に応じて SSL/TLS を利用した接続構成へ移行する。

### 目的
- 認証方式および鍵管理の責任を Docker 側へ寄せる。
- アプリ側設定を簡素化し、接続方式を固定化する。
- 将来の運用統制や本番相当構成への移行をしやすくする。

### 補足
- 現時点では学習優先のため、運用強度よりも構築容易性を優先している。
- 本改修案は、ローカル学習用途から一段進んだ運用設計へ移る際の候補として扱う。
