# 非機能設計

## 1. 性能
- 従業員および派遣先企業の一覧表示・登録・更新・削除の通常操作で実用的な応答時間を維持する。
- 一覧表示は 1 ページ 5 件で取得する。

## 2. 可用性
- DB 接続先は Docker Compose 上で起動する MySQL を前提とする。
- 例外時は処理を中断し、誤った更新継続を防止する。

## 3. ログ
- Hibernate SQL ログを有効化する。
- 障害解析に必要な例外情報を保持する。

## 4. セキュリティ
- 認証・認可は未実装。
- 将来の導入を見据え、Controller 責務を薄く保つ。

## 5. 運用
- 環境設定は `application.properties` で管理する。
- ローカル開発時は Local PC でアプリケーションを起動し、DB は Docker Compose で起動する。
- スキーマ変更は `spring.jpa.hibernate.ddl-auto=update` を前提とする。
