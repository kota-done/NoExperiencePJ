# 基本設計書

## 1. 文書情報
- プロジェクト名: springboot-thymeleaf-crud-web-app
- 作成日: 2026-02-17
- 対象: 従業員情報・派遣先企業情報の CRUD、一覧表示、ページング、ソート機能
- 技術スタック: Java 17 / Spring Boot 3 / Spring MVC / Thymeleaf / Spring Data JPA / MySQL / Maven

## 2. システム概要
- 本システムは、従業員情報と派遣先企業情報を管理する Web アプリケーションである。
- ブラウザから MVC 画面へアクセスし、従業員情報および派遣先企業情報の登録・更新・削除・一覧参照を行う。
- 従業員は所属する派遣先企業と紐づけて管理する。
- 一覧画面ではページングとソートを提供する。
- ローカル開発環境では、Spring Boot アプリケーションは Local PC 上で起動し、MySQL は Docker Compose 上のコンテナを利用する。

## 3. アーキテクチャ方針
- 3 層構成を採用する。
- プレゼンテーション層: Spring MVC Controller + Thymeleaf テンプレート
- アプリケーション層: Service インタフェース/実装
- データアクセス層: Spring Data JPA Repository
- Java アプリケーション内部では、`Employee` 系と `ClientCompany` 系を同じレイヤ構造で管理する。
- ドメインモデルとして `Employee` と `ClientCompany` を保持し、従業員から派遣先企業を参照する構成を前提とする。

## 3.1 実行環境構成
- クライアント:
  - Web ブラウザ（ローカルPC上）
- アプリケーション実行環境:
  - Spring Boot アプリケーション（ローカルPC上）
- データベース実行環境:
  - Docker Compose 管理の MySQL コンテナ

## 4. パッケージ設計
- `com.dev.raise.controller`
  - `EmployeeController`: 画面遷移・リクエスト処理
  - `ClientCompanyController`: 派遣先企業の画面遷移・リクエスト処理
- `com.dev.raise.service`
  - `EmployeeService`: ユースケース定義
  - `EmployeeServiceImpl`: 業務処理実装
  - `ClientCompanyService`: 派遣先企業のユースケース定義
  - `ClientCompanyServiceImpl`: 派遣先企業の業務処理実装
- `com.dev.raise.repository`
  - `EmployeeRepository`: 永続化インタフェース
  - `ClientCompanyRepository`: 派遣先企業の永続化インタフェース
- `com.dev.raise.model`
  - `Employee`: 従業員エンティティ
  - `ClientCompany`: 派遣先企業エンティティ

## 5. 機能設計
### 5.1 F001 従業員一覧表示
- URL: `GET /`
- 処理:
  - 1 ページ目を既定として一覧取得
  - 既定ソート: `firstName` 昇順
  - 従業員氏名、メールアドレス、所属派遣先企業、従業員ステータスを表示する
  - テンプレート `index.html` を返却

### 5.2 F002 従業員登録
- URL: `GET /showNewEmployeeForm`
- 保存 URL: `POST /saveEmployee`
- 処理:
  - 空の `Employee` をモデルに設定
  - 派遣先企業選択肢と従業員ステータスを入力対象とする想定
  - テンプレート `new_employee.html` を返却
- フォーム送信時の処理:
  - フォーム入力を `Employee` にバインド
  - `save` により新規保存を実施
  - 完了後に `/` へリダイレクト

### 5.3 F003 従業員更新
- URL: `GET /showFormForUpdate/{id}`
- 保存 URL: `POST /saveEmployee`
- 処理:
  - ID 指定で対象データを取得
  - 派遣先企業と従業員ステータスを含めて編集可能とする
  - テンプレート `update_employee.html` を返却
  - 送信後は `save` により更新保存し、一覧画面へリダイレクトする

### 5.4 F004 従業員削除
- URL: `GET /deleteEmployee/{id}`
- 処理:
  - ID 指定で削除
  - 完了後に `/` へリダイレクト

### 5.5 F005 従業員一覧ページング/ソート
- URL: `GET /page/{pageNo}?sortField={field}&sortDir={asc|desc}`
- 処理:
  - 1 ページあたり 5 件
  - 指定項目/方向でソート
  - 総件数・総ページ数・現在ページをモデルに設定
  - 一覧表示項目には所属派遣先企業と従業員ステータスを含む

### 5.6 F006 派遣先企業一覧表示
- URL: `GET /client-companies`
- 処理:
  - 登録済み派遣先企業を一覧表示する
  - 企業名、連絡先情報を表示する
  - 更新、削除、新規登録への導線を表示する

### 5.7 F007 派遣先企業登録
- URL: `GET /client-companies/new`
- 保存 URL: `POST /client-companies`
- 処理:
  - 空の `ClientCompany` をモデルに設定する
  - 企業名、連絡先情報を入力する
  - 保存後は派遣先企業一覧へリダイレクトする

### 5.8 F008 派遣先企業更新
- URL: `GET /client-companies/{id}/edit`
- 保存 URL: `POST /client-companies/{id}`
- 処理:
  - 指定 ID の派遣先企業を取得する
  - 更新画面へ既存値を表示する
  - 更新保存後は一覧画面へリダイレクトする

### 5.9 F009 派遣先企業削除
- URL: `GET /client-companies/{id}/delete`
- 処理:
  - 指定 ID の派遣先企業を削除する
  - 完了後に派遣先企業一覧へリダイレクトする

## 6. データ設計
### 6.1 エンティティ: Employee
- テーブル名: `employees`
- カラム:
  - `id`: `BIGINT`, PK, 自動採番 (`IDENTITY`)
  - `first_name`: `VARCHAR`
  - `last_name`: `VARCHAR`
  - `email`: `VARCHAR`
  - `status`: `VARCHAR`
  - `client_company_id`: `BIGINT`, FK

#### 従業員ステータス
- 管理対象の初期値は以下を想定する
  - 出向中
  - 待機
  - 休職中
  - 育成中

### 6.2 エンティティ: ClientCompany
- テーブル名: `client_companies`
- カラム:
  - `id`: `BIGINT`, PK, 自動採番 (`IDENTITY`)
  - `company_name`: `VARCHAR`
  - `contact_email`: `VARCHAR`
  - `phone_number`: `VARCHAR`

### 6.3 エンティティ関連
- `Employee` から `ClientCompany` を参照する。
- 関係は「派遣先企業 1 : N 従業員」を想定する。

## 7. 画面設計
- `index.html`
  - 従業員一覧表示
  - 所属派遣先企業、従業員ステータスを表示
  - ソートリンク、ページリンク、更新/削除ボタン
- `new_employee.html`
  - 新規登録フォーム
  - 派遣先企業選択、従業員ステータス入力を含む想定
- `update_employee.html`
  - 更新フォーム（hidden で `id` 保持）
  - 派遣先企業変更、従業員ステータス変更を含む想定
- `client_company` 系画面
  - 派遣先企業の一覧、登録、更新、削除導線を提供する想定
