# クラス設計メモ

## 全体方針
- 本システムは `Employee` と `ClientCompany` の 2 エンティティで業務を構成する。
- 従業員は派遣先企業へ所属するため、`Employee` から `ClientCompany` を参照する。
- 従業員一覧画面では、従業員情報に加えて所属派遣先企業名と従業員ステータスを表示する。

## EmployeeController
- 従業員管理画面の遷移制御を担当する。
- 一覧、登録画面、更新画面、削除、ページ切替・並び替えを扱う。
- 一覧では所属派遣先企業と従業員ステータスを表示対象に含める。

## ClientCompanyController
- 派遣先企業管理画面の遷移制御を担当する。
- 一覧、登録画面、更新画面、削除を扱う。

## EmployeeService
- 従業員データの業務処理インタフェースを定義する。

## ClientCompanyService
- 派遣先企業データの業務処理インタフェースを定義する。

## EmployeeServiceImpl
- 従業員の取得・保存・削除・ページング処理を実装する。
- 一覧のソート条件を組み立てる。

## ClientCompanyServiceImpl
- 派遣先企業の取得・保存・削除処理を実装する。
- 削除時は従業員の所属有無を確認し、所属従業員がいる場合は削除不可とする想定。

## EmployeeRepository
- `Employee` エンティティの永続化アクセスを担当する。

## ClientCompanyRepository
- `ClientCompany` エンティティの永続化アクセスを担当する。

## Employee
- `employees` テーブルと対応するデータモデルを定義する。
- `EmployeeStatus` を保持し、`ClientCompany` を参照する。

## ClientCompany
- `client_companies` テーブルと対応するデータモデルを定義する。
- `BusinessStatus` を保持する。

## Enum
- 従業員ステータスは `EmployeeStatus` enum で管理する。
- 企業の取引状況は `BusinessStatus` enum で管理する。
